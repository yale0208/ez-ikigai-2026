# 2026創準職志投資自檢
## 概述
創準職志投資自檢根據ikigai模型發展員工的自我評量表，主要透過回顧一年列舉自身的主要貢獻或任務，然後透過自我評鑑該項任務是否擅長?是否喜歡?是否有成就感?薪資回報?
而上述自我評量的指標，分別兩項加總計算，可以分別代表熱情、使命、志向、專業偏向的指標，後續可以針對各分項的平衡來回饋一些指標評價。

## 系統定位
各項指標追求平衡成長，如果特定指標過高則容易有副作用，例如：
- 使命驅動 (Mission-Driven) 副作用： 道德高地帶來的自我壓榨、容易因現實妥協而感到幻滅、過度犧牲個人生活。 
- 熱情驅動 (Passion-Driven) 副作用： 熱情消退時的巨大空虛、為了熱情無視收入或風險、專注度過高導致視野狹窄。 
- 志向驅動 (Calling-Driven) 副作用： 職業與自我認同無法剝離（工作即全部）、將「天職」當作藉口進行不健康的自我犧牲。 
- 專業驅動 (Mastery-Driven) 副作用： 完美主義導致進度拖延、不容許失敗帶來的焦慮、難以與非專業人士溝通（產生傲慢）。 綜合副作用： 長期的極致驅動容易導致身心俱疲（Burnout），無法設定健康界線，使生活重心嚴重失衡。

該系統可以根據設定好的比例公式，回饋相對應的提醒和回饋，像是職場內的心理測驗。

## 系統規格
1. 靜態網站架構僅(vue + html5)，所有資料無須儲存，隨開隨用

## 使用者操作流程（MVP）
1. 列舉去年任務
	- 使用者新增多筆任務（可增刪）。
	- 每筆任務至少包含：任務名稱。
2. 逐筆評分
	- 針對每筆任務評分四項（建議 1~5 分）：
	  - 擅長（Good at）
	  - 喜歡（Love）
	  - 成就感（Meaning）
	  - 薪資回報（Paid）
3. 自動計算與分布展示
	- 系統自動彙總所有任務分數。
	- 依 Ikigai 四象限公式計算四個面向總分。
	- 以雷達圖展示四面向分布。
4. 平衡度評估與回饋
	- 根據四面向分布的平衡程度，顯示對應回饋文字與提醒。

## 資料模型（前端暫存）
- Task
  - id: string
  - name: string
  - scores:
	 - goodAt: number
	 - love: number
	 - meaning: number
	 - paid: number

## 指標計算規則
設每筆任務分數為：
- G = 擅長
- L = 喜歡
- M = 成就感
- P = 薪資回報

每筆任務的四面向分數：
- 熱情 Passion = L + G
- 使命 Mission = L + M
- 志向 Calling = M + P
- 專業 Mastery = G + P

全體加總（N 筆任務）：
- PassionTotal = Σ(L + G)
- MissionTotal = Σ(L + M)
- CallingTotal = Σ(M + P)
- MasteryTotal = Σ(G + P)

## 雷達圖展示規則
- 雷達圖四軸：Passion / Mission / Calling / Mastery。
- 顯示值使用四面向總分。
- 若需視覺一致性，可將四軸做 0~100 正規化：
  - normalized = (value / maxValueAmong4) * 100

## 平衡度與回饋邏輯（MVP）
平衡度判斷採「最大值與最小值差距」：
- gap = max(Passion, Mission, Calling, Mastery) - min(Passion, Mission, Calling, Mastery)

分級建議：
- 平衡：gap <= 10%（相對於四項平均值）
- 輕度失衡：10% < gap <= 25%
- 顯著失衡：gap > 25%

回饋輸出：
1. 先輸出整體平衡等級（平衡 / 輕度失衡 / 顯著失衡）。
2. 再輸出最高面向的副作用提醒（依系統定位定義）：
	- Mission 過高：道德高地自我壓榨、現實落差幻滅、犧牲生活
	- Passion 過高：熱情退潮空虛、忽視收入風險、視野狹窄
	- Calling 過高：工作與自我過度綁定、不健康自我犧牲
	- Mastery 過高：完美主義拖延、焦慮升高、溝通傲慢
3. 若顯著失衡，額外提示 Burnout 風險與界線管理建議。

## 介面需求（MVP）
- 區塊 A：任務列表與新增/刪除
- 區塊 B：任務評分表（四項分數輸入）
- 區塊 C：結果區（四面向總分 + 雷達圖 + 回饋文字）
- 按鈕：一鍵重新計算（或輸入即時更新）
